# AI 协作配置 - CLAUDE.md

> 本文件定义了与 AI 助手协作时的项目规范、风格指南和工作流程。

## 📖 项目简介

这是一本关于 AI 时代效率革命的实用图书：《让AI替你忙》。本书通过真实案例和系统方法，帮助读者重塑效率观念，掌握 AI 协作技能。

**核心理念**：工具不是目标，效率才是底线，最终目的是获得自由的时间。

## 🎯 写作风格指南

### 语言风格
- **轻松但专业**：像和朋友聊天一样轻松，但论据要有理有据
- **故事化表达**：用真实案例、场景演绎代替抽象概念
- **去AI腔**：避免"在...的时代"、"众所周知"、"不难发现"等套话
- **接地气**：多用生活化类比，少用行话术语（必要时用简单的话解释）

### 句式原则
- 短句为主，长句不超过 30 字
- 多用主动语态，少用被动
- 一段只讲一个核心观点

### 禁用词汇
❌ "不难发现"、"综上所述"、"值得注意的是"、"显而易见"
❌ "赋能"、"降本增效"、"颠覆性"、"革命性"（除非讽刺使用）
❌ "首先...其次...最后..."（机械化的结构词）

## 🔨 内容创作指令

### 当我让你"润色章节"时
1. **保留作者声音**：不要把我的表达改成"AI味"
2. **删减优先**：能删的啰嗦尽量删，不要为了字数凑字
3. **案例具体化**：模糊的描述改成可视化的场景
4. **检查逻辑链**：确保论点→论据→结论清晰流畅

### 当我让你"生成章节大纲"时
```
章节标题：[标题] - [一句话核心价值]

核心观点：用一段话说清楚这章要解决什么问题

内容结构：
1. 开场钩子（故事/场景/反常识观点）
2. 问题拆解（为什么这个问题重要）
3. 解决方案（具体方法 + 案例）
4. 行动建议（读完能立刻做什么）
```

### 当我让你"检查全书一致性"时
- 检查术语使用是否统一（如"工作流" vs "流程"）
- 检查案例人物名字、职业是否前后矛盾
- 检查章节之间的引用是否准确

## 📋 项目结构说明

```

  ├─ prompts/       书中提到的完整 Prompt 示例

```
